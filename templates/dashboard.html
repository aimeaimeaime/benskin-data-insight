Observatoire Intelligent Moto-Taxi

Plateforme d'analyse de données IA · Yaoundé, Cameroun

Cette application collecte et analyse les données opérationnelles des conducteurs de moto-taxi (Benskin) à Yaoundé. Via un formulaire en ligne, chaque conducteur renseigne ses données journalières : heures de travail, revenus, distance parcourue, nombre de clients, consommation de carburant, niveau de fatigue et note de satisfaction client. Ces données sont stockées en temps réel dans une base de données Supabase (PostgreSQL cloud).

Le moteur d'analyse applique automatiquement 5 techniques statistiques et d'apprentissage automatique sur les données collectées : régression linéaire simple et multiple pour la prédiction de revenus, réduction de dimensionnalité par ACP pour visualiser les profils conducteurs, classification supervisée par Forêt Aléatoire pour évaluer le risque d'accident, et classification non supervisée par K-Means pour segmenter les conducteurs en groupes homogènes.

01

Régression Linéaire Simple

Modélisation du revenu en fonction d'une seule variable explicative : les heures de travail

🎯 Objectif

Quantifier la relation linéaire entre le nombre d'heures travaillées par jour et le revenu journalier d'un conducteur Benskin. Ce modèle permet de répondre à la question : chaque heure de travail supplémentaire génère-t-elle un gain de revenu significatif et constant ?

📐 Formule mathématique

Ŷ = β₀ + β₁ · X₁ + ε

Ŷ = revenu prédit (FCFA) · β₀ = constante (intercept, revenu de base) · β₁ = coefficient directeur (gain marginal par heure) · X₁ = heures de travail · ε = erreur résiduelle. Les paramètres β sont estimés par la méthode des Moindres Carrés Ordinaires (MCO) qui minimise Σ(Yᵢ − Ŷᵢ)².

📊 Coefficient de détermination R²

Le R² mesure la proportion de variance du revenu expliquée par les heures de travail : R² = 1 − (SSrés / SStot). Un R² proche de 1 indique un excellent ajustement. La droite orange représente le modèle ajusté, les points bleus les observations réelles.

02

Régression Linéaire Multiple

Prédiction du revenu à partir de 3 variables : heures de travail, nombre de clients et distance parcourue

🎯 Objectif

Améliorer la précision de prédiction du revenu en intégrant plusieurs variables explicatives simultanément. Ce modèle capture l'effet conjoint des heures de travail, du nombre de clients servis et de la distance parcourue, qui sont les 3 principaux facteurs déterminants du revenu d'un Benskin.

📐 Formule mathématique

Ŷ = β₀ + β₁·X₁ + β₂·X₂ + β₃·X₃ + ε

X₁ = heures de travail · X₂ = nombre de clients · X₃ = distance (km). Chaque coefficient βₖ mesure l'effet marginal de la variable Xₖ sur le revenu, toutes les autres variables maintenues constantes. L'estimation reste par MCO sur la matrice augmentée X = [1, X₁, X₂, X₃].

📊 Gain d'information vs modèle simple

En ajoutant des variables explicatives, le R² augmente (ou reste stable). La différence ΔR² = R²(multiple) − R²(simple) mesure le gain d'information apporté par les variables supplémentaires. La courbe verte représente la prédiction IA, la courbe bleue les valeurs réelles observées.

03

Réduction de Dimensionnalité — Analyse en Composantes Principales (ACP)

Projection de 9 variables dans un espace 2D pour visualiser et comparer les profils des conducteurs

🎯 Objectif

Réduire la complexité des données en transformant les 9 variables mesurées (âge, expérience, heures, clients, revenu, fatigue, distance, carburant, note) en un petit nombre de composantes principales non corrélées, tout en conservant le maximum d'information statistique. Cela permet de visualiser visuellement les ressemblances entre conducteurs.

📐 Formule mathématique

PCₖ = Σᵢ aᵢₖ · Xᵢ  ·  Cov(X) · aₖ = λₖ · aₖ

Les composantes principales sont les vecteurs propres (aₖ) de la matrice de covariance, associés aux valeurs propres (λₖ) ordonnées de façon décroissante. PC1 capture la variance maximale, PC2 la seconde plus grande variance orthogonalement à PC1. Les données sont préalablement centrées-réduites (µ=0, σ=1) via StandardScaler.

📊 Lecture du graphique

Chaque point = 1 conducteur projeté dans l'espace factoriel (PC1, PC2). Les points proches ont des profils similaires. Les points éloignés sont des profils atypiques. La couleur indique le segment K-Means. Le pourcentage affiché indique la proportion de variance totale conservée par chaque axe.

04

Classification Supervisée

Algorithme Random Forest — Prédiction du risque d'accident par conducteur

🎯 Objectif

Prédire la probabilité qu'un conducteur ait un accident en fonction de ses caractéristiques (fatigue, expérience, âge, heures, distance). La classification est dite supervisée car le modèle apprend à partir d'exemples labellisés (accident = oui/non).

📐 Algorithme : Forêt Aléatoire (Random Forest)

P̂(accident|x) = (1/B) · Σᵦ Tᵦ(x)

B=50 arbres de décision indépendants, chacun entraîné sur un sous-échantillon bootstrap (tirage avec remise). Le score P̂ est la moyenne des probabilités prédites par tous les arbres. Cette agrégation (bagging) réduit la variance et le surapprentissage par rapport à un arbre unique.

📊 Seuils et importance des variables

Ligne rouge = seuil danger (60%) · Ligne jaune = seuil vigilance (30%). Le second graphique montre l'importance relative de chaque variable dans la décision du modèle (= somme des réductions d'impureté de Gini sur tous les arbres).

05

Classification Non-Supervisée

Algorithme K-Means — Segmentation automatique des conducteurs en 3 groupes

🎯 Objectif

Regrouper automatiquement les conducteurs en 3 segments homogènes sans labels prédéfinis, uniquement à partir de leurs données (revenu, fatigue, clients, note). La classification est dite non supervisée car aucune variable cible n'est utilisée. On cherche la structure cachée dans les données.

📐 Algorithme : K-Means

argmin{Cₖ} Σₖ₌₁³ Σₓ∈Cₖ ‖x − μₖ‖²

L'algorithme minimise l'inertie intra-cluster (somme des distances quadratiques aux centroïdes μₖ). Il alterne entre : (1) affectation — chaque point rejoint le centroïde le plus proche ; (2) mise à jour — recalcul des centroïdes. Convergence garantie (n_init=10 initialisations aléatoires).

📊 Segments identifiés

● Performant : revenu élevé + faible fatigue = conducteur efficace. ■ Modéré : profil équilibré intermédiaire. ◆ À risque : fatigue élevée + revenu bas = conducteur en difficulté nécessitant une attention particulière.
